# Инструкция и Опыт Работы Gemini с GitHub (Monorepo: AGword)

Этот документ описывает наши новые правила и мой кристаллизованный опыт взаимодействия с GitHub в условиях единого репозитория (monorepo) для всех подпроектов. 

## 1. Новая Структура (Monorepo)

Вместо разрозненных репозиториев (`.git`) в каждой папке проекта (например, `word/`, `word_tr/` и др.), теперь у нас **один глобальный репозиторий** в корневой папке `D:\AGword`. 

**Преимущества:**
* Единая история коммитов для всех связанных инструментов.
* Один `.gitignore` для настройки исключений.
* Отсутствие путаницы с `submodules` и вложенными `git`-папками.
* Проще делиться всем набором утилит сразу.

## 2. Мои Возможности и Роль (Gemini)

Как AI-ассистент, запущенный в вашей среде, я могу полноценно управлять локальным Git:
* **Инициализация и настройка:** `git init`, `git config` (настраиваю почту и имя бота для коммитов, чтобы не трогать ваши глобальные настройки, если их нет).
* **Управление файлами:** `git add .`, создание `.gitignore`.
* **Коммиты:** `git commit -m "Сообщение"` с детальным описанием того, что я сделал. Я умею формировать атомарные коммиты и описывать изменения.
* **Ветки:** Я могу создавать ветки (`git checkout -b feature-name`), переключаться между ними и подготавливать пул-реквесты, если мы работаем над большой фичей.

## 3. Ограничения (Важно!)

Моя главная слабость — **Авторизация**. 
Когда я пытаюсь выполнить `git push`, GitHub часто запрашивает учетные данные (логин/пароль, токен или подтверждение через браузер Windows). Мои фоновые процессы не имеют интерфейса и не могут нажать "Авторизоваться в браузере". В результате задача зависает.

**Решение (наш алгоритм работы):**
1. Я делаю всю грязную работу локально (пишу код, проверяю, делаю `git add` и `git commit`).
2. Я пишу вам: *"Всё готово и закоммичено! Сделайте пуш."*
3. Вы открываете терминал в `D:\AGword` и делаете короткую команду:
   ```bash
   git push
   ```
4. Если вы настроите SSH-ключи или закешируете учетные данные (Git Credential Manager), то в будущем я, возможно, смогу делать `git push` самостоятельно без зависаний!

## 4. Корневые Файлы

В корне репозитория я создал:
* **`.gitignore`** — глобальный файл, который скрывает `__pycache__`, виртуальные окружения `.venv`, сгенерированные файлы `.docx` и временные файлы среды IDE от попадания в историю коммитов. Это позволяет держать репозиторий чистым.
* **`GEMINI.md`** — этот самый файл, памятка.

## 5. Как мы работаем дальше ("/turboall")

Если вы просите сделать глобальные изменения и "запушить результаты", я буду:
1. Выполнять задачу.
2. Собирать все изменения (`git add .`).
3. Делать коммит (`git commit -m "Auto-commit: [описание]"`).
4. Пробовать выполнить `git push`. Если зависнет — убивать процесс и просить вас выполнить команду руками. (Или, если вы уже залогинены навсегда, всё пройдет гладко).
